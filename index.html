ESP32 API Protocol

ESP32 HTTP REST

This page documents the HTTP REST API contract implemented on each ESP32 robot. In Simulation Mode these endpoints are mocked locally; when Simulation Mode is OFF, the dashboard sends real requests to the configured IP:port.

GET read-only POST state-changing E-STOP safety-critical
POST

/handshake

Connection Setup

Verifies firmware compatibility and retrieves robot capabilities. Must complete before any command is accepted.

Request Body
{
  "controller": "RAL-v2",
  "version":    "2.0.0",
  "timestamp":  1700000000000
}
Response 200
{
  "robotId":         "R-01",
  "firmwareVersion": "v2.4.1-esp32",
  "capabilities":    ["move","turn","autonomy","telemetry"],
  "status":          "ready"
}
Error Codes
400Missing or malformed request body
403Controller version not supported
503Robot not ready (booting)
Handshake must succeed before commands are enabled
Robot marked Unverified and commands disabled on failure
Re-run on reconnect
POST

/command

Command Dispatch

Sends a validated, policy-approved command to the robot. Only dispatched after Policy Engine → Safety Envelope → Handshake chain passes.

Request Body
{
  "commandId": "cmd-abc123",
  "robotId":   "R-01",
  "command":   "MOVE_FORWARD",
  "params":    { "speed": 60 },
  "issuedAt":  1700000000000
}
Response 200 (accepted)
{
  "commandId":  "cmd-abc123",
  "accepted":   true,
  "receivedAt": 1700000000042
}
Response 422 (rejected)
{
  "commandId": "cmd-abc123",
  "accepted":  false,
  "error":     "Speed exceeds firmware limit"
}
Error Codes
400Malformed command payload
401Handshake not verified
422Command rejected by firmware
503Robot busy / not ready
Dashboard polls /command/:id/status every 250–500 ms after acceptance
Command marked Timeout if ack not received in 10 s
GET

/command/:commandId/status

Ack Polling

Polls acknowledgment state after a command is accepted. Dashboard calls every 250–500 ms until completed or failed.

Response 200
{
  "commandId": "cmd-abc123",
  "state":     "executing",
  "timestamp": 1700000000120,
  "error":     null
}
Ack States
receivedRobot received the command
executingRobot is actively executing
completedExecution finished successfully
failedRobot-side execution error
timeoutNo ack within 10 s window

/telemetry

Sensor Data

Returns current robot sensor data. Dashboard polls every 1 s. Shows Telemetry Stale warning if 3+ consecutive requests fail.

Response 200
{
  "robotId":       "R-01",
  "battery":       75,
  "rssi":         -52,
  "speed":         0,
  "movementState": "Idle",
  "uptimeSec":     596
}
Error Codes
503Sensor read failed
408Request timeout
Poll interval: 1000 ms
Telemetry Stale shown after 3 failed polls
GET

/heartbeat

Connectivity Check

Liveness check for the robot. Dashboard calls every 1 s. Robot is marked Disconnected if no response for 5 consecutive seconds.

Response 200
{
  "alive":     true,
  "timestamp": 1700000000000
}
Poll interval: 1000 ms
Mark Disconnected after 5 s no response
Re-run handshake after reconnect
POST

/estop

Safety Critical

Immediately halts all robot motion. Bypasses the normal policy pipeline. Sent directly to robot; no queue delay.

Request Body
{
  "commandId": "estop-xyz789",
  "robotId":   "R-01",
  "issuedAt":  1700000000000
}
Response 200
{
  "commandId": "estop-xyz789",
  "accepted":  true,
  "state":     "stopped"
}
Error Codes
400Malformed payload
503Robot unreachable — physical stop required
Bypasses Policy Engine, Safety Envelope, and Queue
All active and queued commands cancelled
Always logged to Audit Log regardless of mode

About This System

Reydious Autonomy Lab is a mission-control demonstration environment for governed multi-robot systems.

The platform shows how autonomous machines can operate under enforced operational rules. Every command issued by an operator passes through a Policy Engine before a robot executes it.

If a command violates a rule, the system blocks the action and records the decision in the audit log.

This demonstration illustrates governed execution — a system architecture where autonomous machines remain transparent, controllable, and auditable during operation.
